# Community Strategy

## Goals

1. Build a community of developers who build on and contribute to Sentinels.
2. Establish Sentinels as the standard for machine trust infrastructure.
3. Create a sustainable contributor ecosystem that reduces maintainer burden.

## Platforms

| Platform | Purpose | Target Audience |
|---|---|---|
| GitHub | Code, issues, discussions | Developers |
| Discord | Real-time chat, support, community | All |
| Blog | Technical content, updates | Technical decision-makers |
| Twitter/X | Announcements, thought leadership | Industry |
| YouTube | Tutorials, screencasts, conference talks | Learners |
| Newsletter | Monthly updates | Community, customers |

### Roles

| Role | Permissions |
|---|---|
| @everyone | Read #welcome, #general |
| Contributor | All public channels, write access |
| Maintainer | All channels, moderation |
| Core Maintainer | All channels, admin |
| Design Partner | Private channel, priority support |

## Community Events

### Weekly

- **Office Hours** (1 hour, Discord voice): Open Q&A with maintainers

### Monthly

- **Community Call** (1 hour, Zoom): Project updates, demos, guest speakers
- **Contributor Sprint** (virtual): Focused contribution session

### Quarterly

- **Hackathon** (weekend, virtual + local): Themed hacking event
- **Community Survey**: Feedback and priorities

### Annual

- **Sentinel Summit**: In-person conference (Year 2+)

## Moderation Guidelines

### Principles

- Assume good faith
- Be welcoming to newcomers
- Focus on technical merit, not seniority
- No politics, religion, or other divisive topics
- No harassment, trolling, or personal attacks
- No spam or self-promotion

### Enforcement

- First violation: Warning
- Second violation: 7-day mute
- Third violation: Ban

Severe violations (harassment, threats, doxxing) result in immediate ban.

## Community Recognition Program

### Levels

| Level | Badge | Benefits |
|---|---|---|
| Contributor | Discord role, GitHub badge | Early access to features |
| Active Contributor | All above + contributor t-shirt | Priority issue assignment |
| Community Maintainer | All above + maintainer role on GitHub | Voting on roadmap |
| Core Contributor | All above + travel budget for events | Steering committee seat |

### Criteria

- **Contributor**: 1 PR merged or 5 meaningful issue reports
- **Active Contributor**: 10 PRs merged or 3 months of sustained contributions
- **Community Maintainer**: Invitation from core team
- **Core Contributor**: Elected by existing core contributors

## Communication Guidelines

### Blog

- Publish 2 posts per month minimum
- Topics: technical deep dives, tutorials, case studies, architecture decisions
- Tone: technical, honest, precise
- Length: 1,000-2,500 words

### Social Media

- Twitter/X: 3-5 posts per week
- LinkedIn: 1 post per week
- Content: releases, blog posts, community highlights, technical threads

### Newsletter

- Monthly cadence
- Sections: release notes, featured blog post, community spotlight, upcoming events
- Plain text format (no HTML templates)
